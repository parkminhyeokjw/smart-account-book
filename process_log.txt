[STEP 4] DB 설계 도출
---------------------------------------------------------------
- 필요 엔티티 식별:
    · User (사용자)
    · Category (카테고리 — 식비/교통비/주거비 등)
    · Transaction (거래 내역)
    · Receipt (영수증 이미지 + OCR 결과)
    · Budget (월별 예산 한도)
- 관계:
    · User 1:N Category
    · User 1:N Transaction
    · Category 1:N Transaction
    · Transaction 1:N Receipt (선택)
    · User 1:N Budget
- 추천 DB: SQLite (개발/경량) → PostgreSQL (운영/확장)
- ORM 후보: SQLAlchemy (Python) / Prisma (Node.js)

---------------------------------------------------------------
[STEP 5] 자동 입력 로직 설계
---------------------------------------------------------------
- 자동화 소스 4종 정의:
    1. 수동 입력 (UI Form) — 기본 경로
    2. SMS / 카드 알림 파싱 — 정규식 기반 파서 설계
    3. 영수증 OCR — pytesseract / Google Vision 연동 설계
    4. 반복 항목 스케줄러 — schedule 라이브러리, 매일 08:00 실행
- 각 소스의 정확도·구현 난이도 비교표 작성

[STEP 9] schema.sql 생성
---------------------------------------------------------------
- 위치: C:\Users\minjw\smart-account-book\schema.sql
- 적용 방법: mysql -u root -p account_book < schema.sql
- 포함 내용:
    · CREATE DATABASE account_book (utf8mb4)
    · 테이블 6개:
        1. users           — 사용자 (monthly_budget 포함)
        2. categories      — 카테고리 (ENUM type: income/expense)
        3. transactions    — 거래 내역 (ENUM source: manual/auto/sms/ocr)
        4. receipts        — 영수증 OCR 원문 저장
        5. budgets         — 월별 카테고리 예산 한도 (UNIQUE 제약)
        6. recurring_items — Cron 자동 등록 항목 (day_of_month 1~28)
    · 인덱스 3개: idx_tx_user_date, idx_tx_category, idx_tx_source
    · FK 제약: ON DELETE CASCADE / SET NULL 적절히 구분
    · 기본 데이터: 사용자 1명, 카테고리 7개, 반복 항목 3개
